# 2. Related Work

Prior work on phishing detection can be broadly categorized into three approaches: blacklist-based methods, machine learning approaches using hand-crafted features, and deep learning methods.

**Blacklist-based approaches** [4, 5] detect phishing by querying curated databases of known malicious URLs. PhishTank [4] and Google Safe Browsing [5] are the most widely deployed systems in this category. While these approaches achieve high precision for known phishing sites, they suffer from significant detection delays. Since a URL can only be blacklisted after human verification, newly created phishing pages remain undetected for an average of 4–8 hours [3] — a window large enough to victimize thousands of users.

**Machine learning approaches using URL features** have been proposed to address the freshness problem. Garera et al. [6] pioneered URL-based classification using logistic regression on a small set of lexical features. Mohammad et al. [7] extended this line of work with a comprehensive 30-feature set covering URL structure, domain properties, and HTML content, achieving 95.1% accuracy. Jain and Gupta [8] specifically studied URL features for mobile phishing detection, reporting 98.0% accuracy on a dataset of 2,000 URLs. However, these approaches use static, manually defined feature sets that may not generalize to emerging attack patterns, and most are evaluated on small or imbalanced datasets.

**Deep learning approaches** have recently gained traction. Yuan et al. [9] applied LSTM networks directly to raw URL character sequences, eliminating the need for manual feature engineering and achieving 97.3% accuracy. Zhang et al. [10] combined CNNs with attention mechanisms for URL classification, reporting improvements on multilingual phishing detection. While deep learning methods reduce feature engineering effort, they require substantially more training data and computing resources, making them less practical for deployment in resource-constrained environments.

Unlike prior ML-based approaches that rely on manually defined, static feature sets, our work employs a feature importance ranking mechanism that identifies the most discriminative URL features from a pool of 48 candidates. Furthermore, we evaluate PhishGuard on a dataset 15× larger than most prior work, providing stronger evidence of generalizability. To the best of our knowledge, this is the first study to combine feature importance analysis with Random Forest classification for phishing detection on a large-scale balanced dataset.
